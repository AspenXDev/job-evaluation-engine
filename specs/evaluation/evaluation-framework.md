# Evaluation Framework Specification

## Purpose

The evaluation framework interprets capability profiles generated by the lattice scoring model.

While the lattice model produces a multidimensional capability vector for a job role, the evaluation framework provides methods for analyzing and comparing these vectors.

The framework enables consistent evaluation across roles, datasets, and benchmark tasks.

## Inputs

The evaluation framework operates on structured capability profiles produced by the lattice subsystem.

Typical inputs include:

- job capability vectors
- candidate capability vectors
- benchmark task capability profiles

## Core Evaluation Functions

The framework supports several forms of analysis.

### Capability Profile Interpretation

Capability vectors are interpreted to understand the relative strengths and demands of a role.

This allows roles to be characterized by:

- dominant capability dimensions
- balanced vs specialized profiles
- overall capability intensity

### Role Comparison

Capability profiles can be compared across roles.

Comparisons may include:

- similarity between roles
- differences in capability demands
- identification of distinguishing dimensions

### Benchmark Evaluation

Roles can be evaluated against benchmark task datasets.

Benchmark tasks provide reference capability profiles for different task categories such as:

- reasoning-intensive tasks
- planning tasks
- creative tasks

Benchmark evaluation allows the system to contextualize a role within known capability patterns.

### Gap Analysis

Capability profiles can be compared between a candidate and a role.

This analysis identifies:

- capability alignment
- capability gaps
- areas of potential development

## Metrics

The evaluation layer may compute various metrics including:

- capability similarity scores
- dimension distance measures
- benchmark proximity metrics

Exact metric implementations are defined in the implementation layer.

## Outputs

Evaluation results may include:

- structured comparison reports
- capability alignment summaries
- benchmark analysis results

All outputs must conform to the schema defined in:

specs/output-schema.md
