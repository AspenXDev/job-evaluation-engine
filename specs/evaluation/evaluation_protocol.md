# Evaluation Protocol

## Purpose

The evaluation protocol defines the standard procedure for running evaluations using the Job Evaluation Engine.

It ensures that evaluations are conducted in a consistent and reproducible manner across datasets, experiments, and benchmark runs.

## Evaluation Workflow

A standard evaluation run follows the sequence below.

1. Input Preparation
2. Job Parsing and Representation
3. Capability Extraction and Normalization
4. Lattice Scoring
5. Evaluation Analysis
6. Output Generation

## Step 1 — Input Preparation

Inputs are collected in structured form.

Typical inputs include:

- job descriptions
- candidate profiles
- benchmark task datasets

Input templates are defined in:

inputs/

## Step 2 — Job Parsing

The engine parses job descriptions to extract relevant task and capability signals.

Relevant modules:

src/engine/job_parser.py

## Step 3 — Structured Representation

Parsed information is converted into structured job representations.

Relevant modules:

src/engine/job_representation.py

## Step 4 — Lattice Scoring

Structured representations are evaluated using the lattice scoring model to produce capability vectors.

Relevant modules:

src/lattice/

## Step 5 — Evaluation Analysis

Capability profiles are analyzed using evaluation metrics.

This may include:

- role comparison
- benchmark analysis
- candidate-role gap analysis

Relevant modules:

src/evaluation/

## Step 6 — Output Generation

Evaluation results are written as structured outputs.

Outputs conform to the schema defined in:

schemas/evaluation-output.schema.json

Output reports may be generated by scripts located in:

scripts/

## Reproducibility

Evaluation runs should record:

- input dataset
- configuration parameters
- scoring weights
- evaluation metrics used

This ensures that evaluation results can be reproduced and verified.

## Benchmark Execution

Benchmark suites may be executed using:

scripts/run_benchmark_suite.py

Benchmark results are stored in:

outputs/benchmark_results/
